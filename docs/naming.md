# The name, and the naming method

## iladub

Sumerian *íl* ("to lift, carry, deliver, bring forward" — including the accounting
sense "to carry a value forward in a ledger") + *dub* ("clay tablet, document") =
**"the document-carrier."** The name names the *act*, not the artifact: iladub lifts
knowledge out of a human-shaped document and carries it forward into a durable,
machine-readable form. The "carry forward" maps directly onto ET(K)L's Load stage.

Etymology source: J. A. Halloran, *Sumerian Lexicon* v3.0 (`dub` = tablet/document;
*íl* = carrier / to lift, carry, bring forward).

## The naming method (a hard lesson, worth keeping)

Candidates must pass a **three-part bar** before being claimed:
1. **PyPI** free (normalized name — `eduba`, `Eduba`, `edu-ba`, `edu_ba` collide).
2. **GitHub** — no *repo* collision (a bare username handle is only cosmetic).
3. **Web search including "+ ontology / semantic / knowledge graph"** — *no same-domain
   prior art*. This is the check that actually matters.

"Free on PyPI" is necessary but not sufficient. An earlier favorite, `dubsar`, was free
on PyPI but turned out to be a dormant, MIT-licensed, *same-domain* semantic-modeling
project (entity/thing, class/instance) — which would have diluted authorship
provenance. We rejected it on that basis. `eduba` passed the fuller bar but carried an
"edu-" education halo; `iladub` was chosen as the most accurate, distinctive, clean
option and verified free across PyPI, GitHub, npm, crates.io, and .com/.io/.org/.dev.
